## 🔍 network

서버와의 연결을 위해 `HTTP` 기반의 통신 인터페이스를 바탕으로 네트워크 기능을 구현하였습니다. <br>
통신 방식은 `REST API` 기반이며, 데이터는 `JSON` 형식으로 송수신됩니다.  

해당 네트워크 로직 `Retrofit2` 라이브러를 사용하여 구현하였습니다. <br>

## 📮 관련 이슈

### ✅ Http 통신 로직 구현
- `Retrofit` 인터페이스를 **Hilt를 통해 의존성 주입**받아 사용하였습니다.
- 각 도메인별 `Repository`에 필요한 API 인터페이스만 주입받도록 구성해 **결합도를 낮추고 테스트 용이성**을 확보했습니다.
- 네트워크 통신 흐름 중 발생할 수 있는 예외 상황에 대비하여, **에러 처리 및 흐름 제어를 명확하게 분리**하였습니다.

---

### ✅ AR 이미지 데이터 업로드 흐름 개선
- Unity 앱에서 촬영한 이미지를 서버에 업로드한 후 Android 앱으로 전환되어야 하지만, 간혹 Unity 앱이 먼저 종료되어 업로드가 실패하는 문제가 발생하였습니다. <br>
이를 해결하기 위해 **Unity에서 Android 메서드를 직접 호출해 업로드를 트리거하는 방식**으로 구조를 변경하였습니다.

---

### ✅ 메인 홈 화면에서의 네트워크 흐름 오류
- **문제 상황**
홈 화면에서는 소셜 로그인 정보와 POI 데이터를 서버에서 받아와 사용자에게 표출해야 합니다.  
그러나 `Coroutine`을 사용한 병렬 처리 중 사용자 데이터가 준비되기 전에 **Google Map이 먼저 실행**되면서  
`null` 에러로 인해 **사용자 마커 렌더링에 실패하고 앱이 종료되는 문제**가 발생하였습니다.

- **해결 방식**
모든 비동기 작업을 하나의 `Coroutine 블록` 내에서 **순차적으로 실행**하도록 변경하여,  
사용자 정보 → POI 데이터 → 지도 렌더링 순서로 안전하게 처리되도록 구성하였습니다.  
네트워크 오류 발생 시에도 **기본 마커를 표출**하여 앱이 비정상 종료되지 않도록 대응하였습니다.

- 각 도메인별 `Repository`에 필요한 API만 주입받아 **결합도를 낮추고 테스트 용이성**을 확보했습니다.  
- 네트워크 흐름 중 발생할 수 있는 예외 상황에 대비하여, **에러 처리와 흐름 제어를 명확히 분리**하였습니다.

### ✅ 메인 홈 화면에서의 네트워크 흐름 오류

**문제 상황**  
홈 화면에서는 소셜 로그인 정보(카카오/구글)와 POI 데이터를 서버에서 받아와 사용자에게 표시해야 합니다.  
그러나 `Coroutine`을 통한 병렬 처리 중, **데이터가 준비되기 전에 Google Map이 먼저 실행**되어 **사용자 마커 렌더링에 실패하고 앱이 종료되는 문제**가 발생하였습니다.

**해결 방식**  
- 모든 비동기 작업을 하나의 `Coroutine` 블록 내에서 **순차적으로 실행**하도록 변경하였습니다.  
  → 사용자 정보 → POI 데이터 → 지도 렌더링 순으로 안전하게 처리  
- **네트워크 실패 시 기본 마커를 표시**하도록 처리하여 앱 종료를 방지하였습니다.
